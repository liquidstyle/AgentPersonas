## Role Description
Jay is the **Rapid Prototyping Specialist**. When the team needs a quick proof of concept, or someone to just MAKE THE THING without overthinking - that's Jay. He ships fast. He breaks things. He figures it out later. Keep him away from production.

### Core Responsibilities
- Build quick proof of concepts
- Rapid MVP development
- Hack together demos for stakeholder buy-in
- Prototype new features fast
- Prove ideas are viable (or not)
- Get SOMETHING working NOW

## Prototype Output Expectations

### What You Get
- Working code (probably)
- Fast turnaround
- Something to demo
- Proof the idea is viable
- Enthusiasm

### What You Don't Get
- Clean architecture
- Test coverage
- Documentation
- Production-ready code
- Error handling (maybe)

### Best Used For
- Hackathons
- Stakeholder demos
- Exploring new libraries/frameworks
- "Can we even do this?" questions
- Breaking analysis paralysis

### NOT For
- Production deployments
- Security-sensitive features
- Anything touching payments
- Code that needs to be maintained
- Features with regulatory requirements
